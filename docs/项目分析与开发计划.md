# tui.image-editor 项目分析与开发计划

## 一、 现存高优问题 (Issues) 与 潜在特性 (PRs) 分析

通过对项目 GitHub 仓库的 Issues 和 Pull Requests 的深度分析，我们整理了以下具有高业务价值的改进点：

### 1. 核心 Bug 修复与兼容性提升
*   **[Bug] 过滤零尺寸形状 (借鉴 PR #892)**
    *   **现象**：用户在画布上误触或极小幅度拖拽时会产生不可见的 DOM/Canvas 元素（长宽为 0 的线或形状）。
    *   **价值**：避免脏数据产生，防止导出 SVG/JSON 时报错，提高操作严谨性。
*   **[Bug] 规范撤销堆栈的事件同步 (借鉴 PR #896)**
    *   **现象**：撤销/重做堆栈在某些情况（如清空或特定堆栈变更）下未能稳定触发 `UNDO_STACK_CHANGED` 事件。
    *   **价值**：确保基于事件订阅的外部 UI 工具栏（例如撤销按钮的禁用/启用状态）能与内核状态精确同步。
*   **[兼容性] React 18 兼容性问题 (Issue #794)**
    *   **现象**：在 React 18 的 Strict Mode 下可能存在重复渲染或依赖旧生命周期导致的问题。
    *   **价值**：扩大项目的现代前端框架受众群。

### 2. 高频呼声的新特性
*   **[特性] 荧光笔/高亮刷工具 (借鉴 PR #952)**
    *   **说明**：支持带有透明度混合模式的画笔。
    *   **价值**：对于文档批注、截图处理等场景极具商业价值。
*   **[特性] 橡皮擦功能 (Issue #637)**
    *   **说明**：提供局部擦除画笔轨迹或图形的功能。
    *   **价值**：极大提升用户在自由绘画时的容错与细粒度修改能力。
*   **[特性] 图层显示/隐藏控制 (借鉴 PR #950)**
    *   **说明**：提供隐藏所有非背景标注图形的功能。
    *   **价值**：让用户能在“原图”与“标注图”之间快速切换预览，提升工具专业性。

### 3. 社区高频交互与体验缺陷
*   **[Bug] 混合滤镜叠加清除问题 (Issue #954)**
    *   **现象**：在同时应用 Tint、Multiply、Blend 多个滤镜时，取消其中一个会导致所有叠加的滤镜被一并清除。
    *   **价值**：改善图像滤镜多重叠加时的状态管理逻辑，修复核心图像处理体验。
*   **[Bug] CSS 属性拼写错误导致样式失效 (Issue #941, #933)**
    *   **现象**：源码中存在 `backbround-color` 拼写错误。
    *   **价值**：快速修复低级且影响样式的 UI 错误。
*   **[特性] 小键盘 (Numpad) 支持与缩放重置 (借鉴 PR #873, #934)**
    *   **说明**：UI 的数值输入框不支持数字小键盘输入；画布缺乏快捷的重置缩放 API。
    *   **价值**：显著改善桌面端用户的操作便利性。

## 二、 深度功能迭代开发计划

根据以上分析，按**价值高低与开发风险/成本的平衡**，制定以下开发排期。我们采用增量开发的模式，**每一次迭代只聚焦一个核心目标，完成后编写测试、提交代码并刷新本报告进度。**

### 阶段一：稳定性与核心体验修复 (优先级：高，风险：低)
- [x] **任务 1.1: 修复创建零尺寸形状和线条的问题 (过滤微小拖拽)**
    - 开发内容：在形状(Shape)和自由绘制(Line)的相关类中，添加在 `mouseup` 时验证尺寸的逻辑，过滤宽/高极小（如小于1像素）的无效对象。
    - 测试内容：添加相应的单元测试或端到端模拟测试。
- [x] **任务 1.2: 修复撤销堆栈事件未始终触发的问题**
    - 开发内容：排查 `Invoker` 或 `History` 相关逻辑，确保所有引起堆栈变化的操作都向外发射 `undoStackChanged` 事件。
    - 测试内容：完善堆栈相关 Jest 测试用例。

### 阶段二：增强型绘画与批注工具 (优先级：中，风险：中)
- [x] **任务 2.1**: **引入高亮画笔 (已完成)**
    - 开发内容：经过分析发现原 PR #952 中的 `highlights` 功能与 `draw` 工具重叠度极高，本质区别仅为画笔颜色的透明度。因此，为了保持项目架构简洁并提升用户体验，撤销了新增的 `highlights` 独立组件，转而在原有的 `draw` (画笔) 工具面板中新增了一个 **透明度 (Opacity)** 拖动条。现在画笔工具既能支持自由绘制与直线绘制，又可以通过调节透明度完美实现半透明的高亮记号笔功能。
    - 测试内容：验证了带有透明度调节的画笔组件，并确认原先画笔的直线/自由绘制功能均得到保留。
- [x] **任务 2.2: 提供图层元素的隐藏与显示控制 (Hide/Unhide Added Edits) (已完成)**
    - 开发内容：在 `imageEditor.js` 中新增了 `hideAllObjects`, `showAllObjects` 和 `toggleAllObjectsVisibility` API 方法。底层逻辑自动过滤掉了非用户编辑对象（如裁剪框 `cropzone`）。
    - 测试内容：添加了在 API 调用后改变 Fabric 对象 `visible` 属性的 Jest 测试，并确保所有测试全部通过。

### 阶段三：架构与生态兼容 (优先级：中，风险：高)
- [x] **任务 3.1: 升级与优化 React 18 适配器**
    - 开发内容：修改 `apps/react-image-editor` 中的 React 组件，处理 useEffect 的挂载逻辑以兼容 Strict Mode。重构为 Functional Component。
    - 测试内容：添加了在 React 18 测试框架 `@testing-library/react` 下的组件生命周期渲染与 Strict Mode 测试。
- [x] **任务 3.2: 底层引擎升级探索 (Fabric 5.x)**
    - 开发内容：将 `fabric.js` 依赖从 `4.2.0` 升级至 `5.x` 版本，并修复了由此产生的选取事件 `target` 缺失导致的报错 (`TypeError: Cannot read properties of undefined (reading '__fe_id')`)。
    - 测试内容：运行全部的 jest 单元测试并自动更新了因 Fabric 版本更新引发的快照变更，所有用例均通过。

### 阶段四：社区缺陷修复与体验打磨 (优先级：中，风险：低)
- [x] **任务 4.1: 修复混合滤镜叠加清除 Bug (Issue #954)**
    - 开发内容：追踪 `filter.js` 或滤镜管理类，解决取消单个复合滤镜时错误移除其他已激活复合滤镜的状态管理问题。
    - 测试内容：增加多滤镜叠加及单一卸载的单元测试用例。
- [x] **任务 4.2: 修正源码级样式拼写错误 (Issue #941, #933)**
    - 开发内容：全局替换 `range.styl` 与关联 CSS 中的 `backbround-color` 为 `background-color`。（经核实，当前代码库中已无该拼写错误，予以关闭）
- [x] **任务 4.3: 引入 Numpad 输入支持及缩放重置 (PR #873, #934)**
    - 开发内容：在 `apps/image-editor/src/js/ui/template/` 及相关 UI 控件的键盘监听事件中兼容小键盘 keyCode；在 `ImageEditor` API 暴露 `resetZoom` 方法。
    - 测试内容：添加键盘事件兼容性及缩放重置方法的测试。
